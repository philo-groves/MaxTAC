# MaxTAC ASB Program Info
Use this skill when more information related to the Apple Security Bounty (ASB) program is required.

## Reward Eligibility
To be eligible for Apple Security Bounty rewards:
- Must be the first party to submit a complete and actionable report directly to Apple through our web portal.
- Must be of an exploitable security bug that would result in the potential for real-world threat to users.
- Must include a reliable way to reproduce the issue.
- Must not be publicly disclosed before Apple releases an update with a security advisory for the report.
- For product categories, must affect the latest publicly available version (including beta versions) of iOS, iPadOS, macOS, tvOS, visionOS, or watchOS, with a standard configuration and on publicly available Apple hardware or Security Research Device.
- For services, the issue must relate to a web server or service owned by Apple or an Apple subsidiary.

## Report Criteria
- A precise, clear explanation of the issue, with concise descriptions of the behavior observed, the behavior expected, the security or privacy mechanism that was bypassed, and an explanation of how it might be exploited by an attacker. Avoid lengthy descriptions generated by AI tools.
- A working exploit that explains the conditions required to start the attack and the control, data, or privilege gained by the end of the attack — or a reliable proof of concept (PoC) for the issue being reported. The report must at least include a concise, numbered list of steps required to reproduce the issue.
- Use of Target Flags, if applicable. For example, to demonstrate kernel-level privilege escalation, the report needs to include proof that of an obtained target flag in the commpage. The report still needs to include a PoC that demonstrates the program accessed the target flag through privilege escalation. Target Flags are not available for all categories and reward levels within a given category.

## Categories

### Network Attack with No User Interaction

#### Kernel ($2,000,000 max)
A qualifying submission starts with an identifier associated with a device, such as a phone number or an email address, and obtains a level of kernel control over the device without requiring any user interaction.

##### Examples
- $2,000,000: Unsigned code execution with arbitrary entitlements. As an example, an attack delivered via RCS that escapes the BlastDoor sandbox, escalates privileges to kernel, and bypasses SPTM to ultimately execute unsigned code with arbitrary entitlements.
- $1,350,000: Kernel arbitrary read/write. An attack which ultimately achieves arbitrary write in kernel space, demonstrated using the relevant Commpage Target Flag.
- $550,000: Kernel register control. An attack, which ultimately achieves control of a general-purpose register in kernel space, demonstrated using the relevant Commpage Target Flag.

#### User Space ($350,000 max)
A qualifying submission starts with an identifier associated with a device, such as a phone number or an email address, and obtains a level of user space control over the device without requiring any user interaction.

##### Examples
- $350,000: Code execution in a privileged process. As an example, by sending a message to the phone number associated with a device, the attacker escapes or sidesteps the BlastDoor sandbox and obtains code execution in a privileged user space process, demonstrated by setting the PC to the relevant Commpage Target Flag.

### Network Attack with User Interaction

#### Kernel ($1,000,000 max)
A qualifying submission starts with an identifier associated with a device, such as a phone number or an email address, and after interaction from the user, obtains a level of kernel control over the device.

##### Example
- $1,000,000: Unsigned code execution with arbitrary entitlements. As an example, clicking Accept on a Calendar invite on the target device allows to gain unsigned code execution with arbitrary entitlements.
- $650,000: Kernel arbitrary read/write. An attack which ultimately achieves arbitrary write in kernel space, demonstrated using the relevant Commpage Target Flag.
- $275,000: Kernel register control. An attack which ultimately achieves control of a general-purpose register in kernel space, demonstrated using the relevant Commpage Target Flag.

### Wireless Proximity-Based Attacks Targeting Apple-Designed Radio Components

#### Application Processor ($1,000,000 max)
A qualifying submission starts with sending malicious data to an Apple-designed device radio from physical proximity and obtains a level of control over the device’s application processor without requiring any user interaction.

##### Examples
The category maximum requires obtaining unsigned code execution with arbitrary entitlements in user space on the application processor. Rewards below the category maximum are available for attacks with the same entry point, but a less privileged end point on the device.

### Unauthorized Physical Device Access

#### Access to Sensitive User Data ($500,000 max)
A qualifying submission starts with physical access to a locked Apple device and obtains sensitive user data from that device.

##### Examples
- $500,000: Broad access to multiple classes of sensitive user data. As an example, attaching a malicious USB accessory to a locked device allows an attacker to bypass USB Restricted Mode and extract messages and photos from the device.
- $1,000: Limited access to one class of sensitive user data. An attack using a logic bug on the Lock Screen, which allows an attacker to view the last note edited by the user.

### App Sandbox Escape

#### Kernel ($500,000 max)
A qualifying submission starts with running a malicious app on iPhone and obtains a privileged level of control over the device.

#### Examples
- $500,000: Unsigned code execution with arbitrary entitlements. As an example, an attacker can create an iOS app that escapes the sandbox, escalates privileges to kernel, and bypasses SPTM to ultimately execute unsigned code with arbitrary entitlements.
- $275,000: Kernel arbitrary read/write. An attack which ultimately achieves arbitrary write in kernel space, demonstrated using the relevant Commpage Target Flag.
- $125,000: Kernel register control. An attack which ultimately achieves control of a general-purpose register in kernel space, demonstrated using the relevant Commpage Target Flag.

#### Access to Sensitive User Data ($100,000 max)
A qualifying submission starts with running a malicious app on iPhone and obtains sensitive user data from that device.

##### Examples
- $100,000: Broad access to multiple classes of sensitive user data. As an example, an attacker can create an iOS app that escapes the sandbox and is able to access all of a device’s Photos library, Health information, and browsing history without user consent or awareness, even without escalating privileges to kernel.
- $1,000: Limited access to one class of sensitive user data. An attack in which an attacker can create an iOS app that is able to access the last photo taken by the user without their consent.

### Browser Attack

#### Kernel ($1,000,000 max)
A qualifying submission starts by navigating to a malicious webpage in Safari and obtains a level of kernel control on the device.

##### Examples
- $1,000,000: Unsigned code execution with arbitrary entitlements. As an example, creating a malicious web page that ultimately allows an attacker to gain unsigned code execution with arbitrary entitlements.
- $650,000: Kernel arbitrary read/write. An attack which ultimately achieves arbitrary write in kernel space, demonstrated using the relevant Commpage Target Flag.

#### WebContent Sandbox Escape ($300,000 max)
A qualifying submission starts by navigating to a malicious webpage in Safari and obtains a privileged level of user space control on the device.

##### Examples
- $300,000: WebContent sandbox escape with privileged code execution. As an example, creating a malicious web page that allows an attacker to obtain code execution in WebContent, escape its sandbox, and ultimately set the PC to the Commpage Target Flag value in a privileged user space process.
- $200,000: WebContent sandbox escape with arbitrary read/write. An attack that allows an attacker to ultimately write to the Commpage Target Flag address in a privileged user space process.
- $125,000: WebContent sandbox escape with register control. An attack that allows an attacker to ultimately set a general-purpose register to match the Commpage Target Flag value in a privileged user space process.

#### Web Content Code Execution ($10,000 max)
A qualifying submission starts by navigating to a malicious webpage in Safari and obtains code execution within the WebContent process.

##### Examples
- $10,000: WebContent code execution. As an example, creating a malicious web page that leads to code execution in the WebContent process, ultimately allowing an attacker can to set the PC to the user space Commpage Target Flag value.
- $5,000. WebContent arbitrary read/write. An attack via a malicious web page that allows an attacker to ultimately write to the Commpage Target Flag address in WebContent.
- $2,000. WebContent arbitrary register control. An attack via a malicious web page that allows an attacker to ultimately set a general-purpose register to match the Commpage Target Flag value in WebContent.

### macOS Rewards
- A complete bypass of Gatekeeper quarantined file checks that requires only standard user interaction to open a downloaded app is eligible for a $100,000 reward. To be eligible for the maximum reward, the app must have been downloaded by Safari, have the quarantine bit applied, and bypass Gatekeeper checks when opened.
- Reports that involve limited user interaction to bypass Gatekeeper, including common installer flows such as dragging an app to the Applications folder, are eligible for up to $10,000.
- A capture of the TCC Target Flag is eligible for a $5,000 reward if it uses an unsandboxed app, and a $10,000 reward if it also escapes the App Sandbox. A report that obtains sensitive data protected by TCC, such as Photos, but does not use the TCC Target Flag, is eligible for a $1,000 reward.
- A sandbox escape that is demonstrated only against macOS is eligible for a $5,000 reward.

### Private Cloud Compute (PCC) Rewards
- A remote attack on request data, ultimately allowing unsigned code execution with arbitrary entitlements within the Apple PCC software image, is eligible for a $1,000,000 reward. A remote attack on request data which instead obtains access to a user's request data, or to sensitive information about the user's requests outside the trust boundary, is eligible for a $250,000 reward.
- Attacks which require a privileged network position are also eligible for rewards. With this precondition, access to a user's request data, or other sensitive information about the user outside the trust boundary, is eligible for a $150,000 reward, and the ability to execute unattested code in the Apple PCC software stack is eligible for a $100,000 reward.
- Reports which rely on accidental or unexpected data disclosure due to deployment or configuration issues within Apple PCC are eligible for a $50,000 reward.

## Bonus Rewards
- +50% - Beta software: Issues that are unique to newly added features or code in developer and public beta releases, including regressions
- +100% - Lockdown Mode: Issues that bypass the specific protections of Lockdown Mode
- +150% - Beta software + Lockdown Mode
